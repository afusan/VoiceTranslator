# claudeKnowledge — エージェント別メモリ

各サブエージェント(および必要に応じてオーケストレータ)が、セッションをまたいで持ち越す
**抽象化された教訓(lesson)** を置く場所。`.claude/agents/` の「エージェント定義」とは分離し、
ここには「定義に対する可変の学び」を蓄積する。

## 構成

```
claudeKnowledge/
└── <agentName>/
    └── memory.md   … そのエージェント専用のメモリ(教訓の蓄積)
```

## 規律

- **起動時**: 各エージェントは自分の `memory.md` を読み、過去の教訓を今回の作業に活かす。
  - **メンテも同時に行う**: 既存教訓と現状を突き合わせ、**事実と乖離している記述は削除・修正してから着手**する
    (腐ったメモリを放置しない)。リネーム済みの仕組み・既に解消した落とし穴・古いプロジェクト方針への
    言及などはこのタイミングで掃除する。
- **終了時**: その作業で得た再現性のある教訓を、最後の行動として `memory.md` に追記する。
  - サブエージェントには「自分が閉じる瞬間」を制御するイベントが無いため、書き込みは
    各エージェントの**最終行動**として行う(将来は `SubagentStop` / `SessionEnd` フックで機械化する)。
- **抽象度を保つ**: 残すのは「繰り返す落とし穴・プロジェクト/利用者の好み・有効な勘所」。
  コードや git が既に持つ具体(ファイル名・行番号・個別の修正内容)は残さない。
- **書き込み範囲**: 各エージェントが書いてよいのは自分の `memory.md`(と、役割上の成果物。
  例: designReviewer は対象の `designReview.md`)に限る。

## git 管理について

このディレクトリは **git 管理対象**(教訓を版管理・レビューしたいため)。
作業用の一時領域 `tmp/`(管理対象外)とは性格が異なる。

## 現在のメモリ

- `designReviewer/` — 設計レビュアーの教訓
- `worker/` — 実装者の教訓
- `reviewer/` — 最終レビュアーの教訓
