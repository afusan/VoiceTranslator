Architecture (アーキテクチャ)

本ドキュメントは レイヤ構成 / 各レイヤの役割 / スレッド構成 / レイヤ間でやり取りするデータ を定義する。
クラス・モジュール単位の詳細は Class.md を参照。

1. 全体レイヤ図

本アプリは「音声を取り込み → 翻訳して → 別デバイスに再生する」までを、 役割ごとに分割した6つのレイヤで構成する。各レイヤは抽象I/Fと1つの具象実装で構成し、後から差し替え可能とする(MVPでは各1実装のみ)。

2. 各レイヤの役割

レイヤ役割
AudioCapture 選択された入力デバイスから音声を取得し、内部標準フォーマット(16kHz/mono/float32)に正規化してパイプラインに供給する。
VAD 連続するPCMストリームから発話区間を検出し、無音で区切って「1発話分のPCM」として下流に流す。
ASR 発話単位の PCM を入力言語のテキストに書き起こす。翻訳は行わない。
Translator 書き起こしテキストを src言語 → tgt言語 に翻訳する。
TTS 翻訳テキストを音声(PCM)に合成する。
AudioOutput 合成音声を、入力とは別の出力デバイスで再生する。

横断機能

機能役割
AppControllerGUI と内部モジュールの仲介。バックエンド初期化、モデル状態の発信、設定操作の窓口、ドロップ通知の受け口を担う。
PipelineCoordinator各レイヤの起動・停止と、レイヤ間キューの管理を担う。
UtteranceLedger発話ごとのメタ情報(タイミング・言語・テキスト・状態)を中央で集約する。最終段で取り出して履歴ログに書く。スレッドセーフ。
SequenceGenerator発話ごとに一意の通し番号を発行する。各ログにこの番号が載り、レコード間の対応を取れる。
ConfigStore選択中のバックエンド、デバイス、言語ペア、ログ出力先などの設定値を YAML で永続化する。
Loggerアプリ全般のログを stdout とファイルに出力する。
TextLogger翻訳前 / 翻訳後 テキストの個別ファイル追記。ON/OFF は src/tgt 個別に切り替え可能。
JsonlLogger翻訳1件を1行のJSONとして履歴ファイルに追記する(機械処理向け)。
ErrorHandler例外を 致命 / 回復可能 / スキップ可能 / 警告 の4分類で振り分ける。handle(exc, *, stage, seq_id) で「どの段で・どの発話で」起きたかをログとコールバックに伝える。NotificationThrottle 注入で同種エラー連発時の UI 通知を集約・抑制可能。詳細は ErrorCatalog.md を参照。
NotificationThrottleUI 通知の集約・抑制。(stage, 例外型) をキーに時間窓で rate limit し、抑制された件数を次の通知に乗せる。ログは抑制しない。
DeviceValidator入力デバイスと出力デバイスが同一でないことを起動時に保証する。フィードバック防止のため。
cache_check各バックエンドのモデルがローカルキャッシュにあるかを軽量に判定する。GUI の初期状態表示に使う。
GUI設定変更・動作開始停止・直近状態・モデルステータスを可視化する。

3. スレッド構成

各レイヤを独立スレッドで動かし、UI と取得・処理・再生を分離する。各スレッドはキューで連結し、後段が詰まっても前段が止まらない。

スレッド役割該当クラス/関数
UI 画面更新とユーザ操作を受け付ける。常に応答可能を維持する。 tkinter mainloop / MainWindow
Loader 各レイヤのバックエンドを順次初期化する。完了/失敗を UI に通知する。1回限り起動。 AppController._loader_body
Input 入力音声を取り込み、発話単位に切り分けて下流に渡す。 PipelineCoordinator._input_loop
ASR 発話を書き起こす。 PipelineCoordinator._asr_loop
Translator 書き起こしを翻訳する。 PipelineCoordinator._translator_loop
TTS 翻訳結果を音声に合成する。 PipelineCoordinator._tts_loop
Output 合成音声を再生し、発話の完了を通知する。 PipelineCoordinator._output_loop

キューあふれ時の挙動

  • キューが上限に達した状態で put すると、最古の要素を捨てて新しいものを優先する。リアルタイム性を確保するため。
  • 捨てられた発話は on_dropped(seq_ids, stage_name) コールバック + WARN ログで通知する。
  • テキストログは各段で直書きされるため、再生されなくても src/tgt テキストは残る。

停止シーケンス

  1. UI 停止ボタン → AppController.stop_pipeline()
  2. PipelineCoordinator が stop_event をセット
  3. 各スレッドのループ先頭で stop_event.is_set() をチェック → True なら新規 get せずに抜ける。処理中の発話は完了させてから終了。
  4. 各キューにセンチネルを順序投入 → 全スレッド join
  5. capture.stop() / output.stop() を呼んでデバイス解放
  6. 再 start() 時は全キュー drain + ledger.clear() で残骸を引きずらない

4. レイヤ間でやり取りする情報

キューを流れるのは seq_id + 次段に必須な最小データ。メタ情報(timeline / lang / text 等)は UtteranceLedger に中央保管され、各スレッドが seq_id をキーに書き込み / 読み出しを行う。

4-1. キューを流れる payload(各段)

キュー 送信元 → 受信先 payload 型 含まれる情報
captured_queue Input → ASR RawPayload seq_id, pcm(np.ndarray 16kHz/mono/float32), src_lang_hint("auto" or 言語コード)
recognized_queue ASR → Translator TranscribedPayload seq_id, src_text(str), src_lang(検出済み)
pcm は載せない(ASR後不要)
translated_queue Translator → TTS TranslatedPayload seq_id, tgt_text(str), tgt_lang
src_text は載せない(ledger に保管済み)
synthesized_queue TTS → Output SynthesizedPayload seq_id, tts_pcm(np.ndarray), tts_samplerate

4-2. UtteranceLedger に蓄積されるメタ情報

seq_id をキーに、以下の情報を発話単位で1つのレコードに集約する。

カテゴリ内容用途
進行タイミング t_capture, t_vad_end, t_asr, t_translate, t_tts, t_playback(各ステージの完了時刻) レイテンシ計算、jsonl の timeline
言語 src_lang(ASR検出 or hint)、tgt_lang(設定値) text-log の [lang] 表示、jsonl
テキスト src_text(ASR出力)、tgt_text(Translator出力) jsonl 本文(text-log は各段で直書き)
状態 completed / skipped / dropped / failed の遷移と該当ステージ 診断、jsonl への状態記録、UI 通知
(任意)メタ 使用バックエンド名、入力音声秒数 等 切替テスト時の比較。MVPでは省略可

4-3. バックエンドI/F

バックエンドは内部標準データ型を知らず、入出力はプリミティブのみで扱う。

レイヤ抽象 I/F シグネチャ
AudioCapturelist_sources() / start(source_id) / read_chunk(timeout) -> np.ndarray | None / stop()
VADprocess(chunk: np.ndarray) -> list[VadSegment] / reset()
ASRtranscribe(pcm: np.ndarray, src_lang_hint: str) -> tuple[str, str]
Translatortranslate(src_text: str, src_lang: str, tgt_lang: str) -> str
TTSsynthesize(text: str, lang: str) -> tuple[np.ndarray, int]
AudioOutputlist_devices() / start(device_id) / play(pcm: np.ndarray, samplerate: int) / stop()

5. データフロー(発話1件あたり)

  1. Input: PCMチャンク連続取得 → VAD が無音で区切り → seq_id = SequenceGenerator.next() → ledger.init(seq_id, t_capture, t_vad_end, src_lang_hint) → captured_queue.put(RawPayload(seq_id, pcm, hint))
  2. ASR: captured_queue から取出 → (src_text, src_lang) = asr.transcribe(pcm, hint) → ledger.record(seq_id, t_asr=now, src_text, src_lang) → (任意) text_logger.write_src(seq_id, src_text, src_lang) → recognized_queue.put(TranscribedPayload(seq_id, src_text, src_lang))
  3. Translator: recognized_queue から取出 → tgt_text = translator.translate(src_text, src_lang, tgt_lang) → ledger.record(seq_id, t_translate, tgt_text, tgt_lang) → (任意) text_logger.write_tgt(seq_id, tgt_text, tgt_lang) → translated_queue.put(TranslatedPayload(seq_id, tgt_text, tgt_lang))
  4. TTS: translated_queue から取出 → (pcm, sr) = tts.synthesize(tgt_text, tgt_lang) → ledger.record(seq_id, t_tts) → on_text_ready(ledger.peek(seq_id))(UI 履歴の前倒し通知。音より先にテキストを出す) → synthesized_queue.put(SynthesizedPayload(seq_id, pcm, sr))
  5. Output: synthesized_queue から取出 → output.play(pcm, sr) → ledger.record(seq_id, t_playback) → record = ledger.pop(seq_id) → jsonl.write(record) → on_utterance_done(seq_id, record)(レイテンシ確定通知)

6. ログとの対応(seq_id による横断追跡)

全ログレコードに seq_id=N を含める。grep で対応が取れる。

app.log:
[2026-05-27 12:34:56] [INFO] seq=42 ASR完了 src="hello world"
[2026-05-27 12:34:57] [WARNING] seq=42 queue overflow in synthesized_queue (dropped)

soundsrc.txt:
[2026-05-27 12:34:56] [seq=42] [en] hello world

translated.txt:
[2026-05-27 12:34:56] [seq=42] [ja] こんにちは世界

translations.jsonl:
{"seq_id": 42, "src_text": "hello world", "tgt_text": "こんにちは世界", "src_lang": "en", "tgt_lang": "ja", "latency_ms": 850, "timeline": {...}}

7. 拡張ポイント

  • 新しいバックエンドを追加するときは、対応する抽象I/Fを実装し BackendRegistry に登録するだけでGUIに出現する。
  • OS別の音声取得は AudioCapture の差し替えで対応(Windows = soundcard、Linux = GStreamer 等)。
  • per-app 音声取得は OS別バックエンドの一部として将来追加(pendList参照)。
  • ローカルLLM翻訳は Translator の追加実装で対応(例: Ollama)。
  • バックエンド固有パラメータは backends_config.<name>.* 名前空間で config.yaml から渡す(SAPI rate 等が既に実装済)。

8. 横断方針

  • 役割の表明: 各クラスのコメント/Docstring 冒頭に1〜2行で役割を明記する(CLAUDE.md 準拠)。
  • エラー: 各バックエンドは内部例外を AppError(severity 付き) に包んで送出。中央で ErrorHandler が4分類に振り分ける。
  • 計測: 全ステージで ledger.mark_time(seq_id, stage) を呼ぶ。
  • デバイス分離: 起動時に入力デバイス ≠ 出力デバイスを保証(同一なら起動拒否)。AEC は将来オプション。
  • シーケンスID: 全発話に一意IDを発行し、すべてのログに付与してレコード間追跡を可能にする。
  • 停止/再開のクリーンさ: stop_event ベースの協調停止、再start時にキュー + ledger を drain。
