Architecture (アーキテクチャ)

本ドキュメントは レイヤ構成 / スレッド構成 / 各レイヤの役割 / レイヤ間でやり取りするデータ(I/F) を定義する。
クラス・モジュール単位の詳細は Class.md を参照。
※ 改訂 2026-05-27: Process スレッドを ASR/Translator/TTS の独立3スレッドに分割し、5スレッド構成に変更。データ伝達を最小化し、中央管理 UtteranceLedger を導入。

2. スレッド構成(B+案 → 案C 完全並列化)

スレッド役割入力出力
UI (main) tkinter mainloop。ユーザ操作と画面更新のみ。常に応答可能。 ユーザ操作イベント AppController への呼び出し
Loader (1回限り) 各レイヤのバックエンドを順にインスタンス化(モデルロード)。完了/失敗を UI に通知。 start 指示 5スレッド起動 / on_started or on_failed
Input thread capture.read_chunk → vad.process。発話確定で seq_id 発行 → ledger 初期化 → RawPayload を q_raw に投入。 マイク/LB 音声(連続) q_raw への RawPayload
ASR thread q_raw から取出 → asr.transcribe(pcm, hint) → ledger に t_asr / src_text / src_lang を記録 → (任意で) text-log.write_src → TranscribedPayload を q_tr に投入。 q_raw から RawPayload q_tr への TranscribedPayload
Translator thread q_tr から取出 → translator.translate(text, src, tgt) → ledger に t_translate / tgt_text / tgt_lang を記録 → (任意で) text-log.write_tgt → TranslatedPayload を q_xl に投入。 q_tr から TranscribedPayload q_xl への TranslatedPayload
TTS thread q_xl から取出 → tts.synthesize(text, lang) → ledger に t_tts 記録 → SynthesizedPayload を q_syn に投入。 q_xl から TranslatedPayload q_syn への SynthesizedPayload
Output thread q_syn から取出 → output.play(pcm, samplerate) → ledger に t_playback 記録 → ledger.pop(seq_id) で全情報回収 → jsonl 書込 + on_utterance_done コールバック。 q_syn から SynthesizedPayload 音声再生 + jsonl + UI通知

キューあふれ時の挙動(全段共通)

  • キューが上限に達した状態で put すると、最古の要素を捨てて新しいものを優先する(リアルタイム性確保)。
  • 捨てられた発話は on_dropped(seq_ids, stage_name) コールバック + WARN ログ(seq_id 付き)で通知。
  • AppController は dropped を text-log に転送(部分的にでも記録を残す)。

停止シーケンス

  1. UI 停止ボタン → AppController.stop_pipeline()
  2. PipelineCoordinator が stop_event をセット
  3. 各スレッドのループ先頭で stop_event.is_set() をチェック → True なら新規 get せずに抜ける。すでに処理中の発話は完了させてから終了。
  4. 各キューにセンチネルを順序投入 → 全スレッド join
  5. capture.stop() / output.stop() を呼んでデバイス解放
  6. 再 start() 時は: 全キュー drain + ledger.clear() で残骸を引きずらない

3. 各レイヤの役割(レイヤ単位)

レイヤ役割
AudioCapture 選択された入力デバイスから音声を取得し、内部標準フォーマット(16kHz/mono/float32)に正規化してパイプラインに供給する。
VAD 連続するPCMストリームから発話区間を検出し、無音で区切って「1発話 = 1 RawPayload」として下流に流す。
ASR 発話単位の PCM を入力言語のテキストに書き起こす。出力先言語の指定はしない(=翻訳しない)。
Translator 書き起こしテキストを src言語 → tgt言語 に翻訳する。
TTS 翻訳テキストを音声(PCM)に合成する。
AudioOutput 合成音声を、入力とは別の出力デバイスで再生する。

横断機能

機能役割
AppControllerGUI と内部モジュールの仲介。Loader スレッド起動 / バックエンド生成 / モデル状態の発信 / 設定操作の窓口 / drop ハンドラ。
PipelineCoordinator5スレッドの起動・停止、4キューの管理、UtteranceLedger との連携。
UtteranceLedgerseq_id をキーとした発話メタ情報の中央管理。各スレッドから timeline / lang / text / 状態 を積み、最終段で jsonl 出力時に pop される。スレッドセーフ。
SequenceGenerator発話ごとに一意の seq_id を発行(1, 2, 3, ...)。全ログ(app.log / jsonl / soundsrc.txt / translated.txt)に付与され、レコード間の対応を取れる。
ConfigStore選択中のバックエンド、デバイス、言語ペア、ログ出力先、各バックエンド固有パラメータ等を YAML で永続化。
Loggerアプリ全般のログ(stdout + app.log)。各メッセージに seq_id を含めることで横断追跡可能。
TextLogger翻訳前 / 翻訳後 テキストの個別ファイル追記。write_src(seq_id, text, lang) と write_tgt(seq_id, text, lang) に分離。ON/OFF 個別。
JsonlLogger翻訳1件 = jsonl 1行(機械処理向け)。Output 完了時に ledger.pop で全情報を取得して書く。
ErrorHandler例外を 致命 / 回復可能 / スキップ可能 / 警告 の4分類で振り分ける。詳細は ErrorCatalog.md(未作成)。
DeviceValidator起動時に「入力デバイス ≠ 出力デバイス」を保証(フィードバック防止)。
cache_check各バックエンドのモデルがローカルキャッシュにあるかを軽量判定。GUI の初期ステータス表示に使う。
GUI設定変更・動作開始停止・直近状態 + モデルステータスの可視化を提供する(customtkinter)。

4. レイヤ間でやり取りする情報(I/F)

キューを流れるのは seq_id + 次段に必須な最小データ。メタ情報(timeline / lang / text 等)は UtteranceLedger に中央保管され、各スレッドが seq_id をキーに書き込み / 読み出しを行う。

4-1. キューを流れる payload(各段)

キュー 送信元 → 受信先 payload 型 含まれる情報
q_raw Input → ASR RawPayload seq_id, pcm(np.ndarray 16kHz/mono/float32), src_lang_hint("auto" or 言語コード)
q_tr ASR → Translator TranscribedPayload seq_id, src_text(str), src_lang(検出済み)
pcm は載せない(ASR後不要)
q_xl Translator → TTS TranslatedPayload seq_id, tgt_text(str), tgt_lang
src_text は載せない(ledger に保管済み)
q_syn TTS → Output SynthesizedPayload seq_id, tts_pcm(np.ndarray), tts_samplerate

4-2. UtteranceLedger に蓄積されるメタ情報

seq_id をキーに、以下の情報を発話単位で1つのレコードに集約する。

カテゴリ内容用途
進行タイミング t_capture, t_vad_end, t_asr, t_translate, t_tts, t_playback(各ステージの完了時刻) レイテンシ計算、jsonl の timeline
言語 src_lang(ASR検出 or hint)、tgt_lang(設定値) text-log の [lang] 表示、jsonl
テキスト src_text(ASR出力)、tgt_text(Translator出力) jsonl 本文(text-log は各段で直書き)
状態 completed / skipped / dropped / failed の遷移と該当ステージ 診断、jsonl への状態記録、UI 通知
(任意)メタ 使用バックエンド名、入力音声秒数 等 切替テスト時の比較。MVPでは省略可

4-3. バックエンドI/F(プリミティブ化)

バックエンドは Utterance 型を知らず、入出力はプリミティブのみ。テスト容易性 + 内部依存削減。

レイヤ抽象 I/F シグネチャ
AudioCapturelist_sources() / start(source_id) / read_chunk(timeout) -> np.ndarray | None / stop()
VADprocess(chunk: np.ndarray) -> list[np.ndarray] / reset()
(発話確定のPCM群を返す。seq_id 発行は Input thread 側)
ASRtranscribe(pcm: np.ndarray, src_lang_hint: str) -> tuple[str, str]
戻り値: (src_text, detected_lang)
Translatortranslate(src_text: str, src_lang: str, tgt_lang: str) -> str
TTSsynthesize(text: str, lang: str) -> tuple[np.ndarray, int]
戻り値: (pcm, samplerate)
AudioOutputlist_devices() / start(device_id) / play(pcm: np.ndarray, samplerate: int) / stop()

5. データフロー(発話1件あたり)

  1. Input: PCMチャンク連続取得 → VAD が無音で区切り → seq_id = SequenceGenerator.next() → ledger.init(seq_id, t_capture, t_vad_end, src_lang_hint) → q_raw.put(RawPayload(seq_id, pcm, hint))
  2. ASR: q_raw から取出 → (src_text, src_lang) = asr.transcribe(pcm, hint) → ledger.record(seq_id, t_asr=now, src_text, src_lang) → (任意) text_logger.write_src(seq_id, src_text, src_lang) → q_tr.put(TranscribedPayload(seq_id, src_text, src_lang))
  3. Translator: q_tr から取出 → tgt_text = translator.translate(src_text, src_lang, tgt_lang) → ledger.record(seq_id, t_translate, tgt_text, tgt_lang) → (任意) text_logger.write_tgt(seq_id, tgt_text, tgt_lang) → q_xl.put(TranslatedPayload(seq_id, tgt_text, tgt_lang))
  4. TTS: q_xl から取出 → (pcm, sr) = tts.synthesize(tgt_text, tgt_lang) → ledger.record(seq_id, t_tts) → q_syn.put(SynthesizedPayload(seq_id, pcm, sr))
  5. Output: q_syn から取出 → output.play(pcm, sr) → ledger.record(seq_id, t_playback) → record = ledger.pop(seq_id) → jsonl.write(record) → on_utterance_done(seq_id, record)

6. ログとの対応(seq_id による横断追跡)

全ログレコードに seq_id=N を含める。grep で対応が取れる。

app.log:
[2026-05-27 12:34:56] [INFO] seq=42 ASR完了 src="hello world"
[2026-05-27 12:34:57] [WARNING] seq=42 queue overflow in q_syn (dropped)

soundsrc.txt:
[2026-05-27 12:34:56] [seq=42] [en] hello world

translated.txt:
[2026-05-27 12:34:56] [seq=42] [ja] こんにちは世界

translations.jsonl:
{"seq_id": 42, "src_text": "hello world", "tgt_text": "こんにちは世界", "src_lang": "en", "tgt_lang": "ja", "latency_ms": 850, "timeline": {...}}

7. 拡張ポイント

  • 新しいバックエンドを追加するときは、対応する抽象I/Fを実装し BackendRegistry に登録するだけでGUIに出現する。
  • OS別の音声取得は AudioCapture の差し替えで対応(Windows = soundcard、Linux = GStreamer 等)。
  • per-app 音声取得は OS別バックエンドの一部として将来追加(pendList参照)。
  • ローカルLLM翻訳は Translator の追加実装で対応(例: Ollama)。
  • バックエンド固有パラメータは backends_config.<name>.* 名前空間で config.yaml から渡す(SAPI rate 等が既に実装済)。

8. 横断方針

  • 役割の表明: 各クラスのコメント/Docstring 冒頭に1〜2行で役割を明記する(CLAUDE.md 準拠)。
  • エラー: 各バックエンドは内部例外を AppError(severity 付き) に包んで送出。中央で ErrorHandler が4分類に振り分ける。
  • 計測: 全ステージで ledger.mark_time(seq_id, stage) を呼ぶ。
  • デバイス分離: 起動時に入力デバイス ≠ 出力デバイスを保証(同一なら起動拒否)。AEC は将来オプション。
  • シーケンスID: 全発話に一意IDを発行し、すべてのログに付与してレコード間追跡を可能にする。
  • 停止/再開のクリーンさ: stop_event ベースの協調停止、再start時にキュー + ledger を drain。
