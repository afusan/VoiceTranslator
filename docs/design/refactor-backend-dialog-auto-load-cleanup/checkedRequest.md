# checkedRequest

## 目標

バックエンドの「設定」ボタンから開くダイアログに残っている「自動時に自動ロード」
項目を、UI / ロジック / 設定スキーマ / i18n / テストにわたって**完全に削除**する。
過去に「不要」と決定したものの削除が徹底できておらず、
ダイアログ上に死んだスイッチや関連する設定キー・分岐が残っている状態を解消する。

削除後はダイアログを開いても当該項目が表示されないこと、設定ファイルからキーが
消えること、関連する自動ロード分岐コードがコードベースから消えていることを満たす。

## スコープ

- バックエンド設定ダイアログ(Backend 系 Panel から開く設定ダイアログ)の UI
- 当該項目が制御していた `Settings` / `ConfigStore` 上のキー
- 当該キーを読んでいた logic(自動時に自動ロードする分岐)
- i18n カタログ(`gui/i18n.py`)上の対応キー
- 関連テスト(該当項目を検証していた small/middle テストの修正・削除)

## 非目標

- 他の自動ロード/ lazy ロード機構の挙動変更
- AuthState 表示・差分リロード等、`branch-lazy-load-auth-gate` で確定済みの設計の見直し
- ダイアログのその他項目のリファクタ・整理
- master へのマージ(マージは利用者の明示指示があってから)

## 起点ヒント

- 関連メモリ: `branch-lazy-load-auth-gate`(2026-06-11 マージ済)
  「変更即ロード廃止 / AuthState 静的表示 / 差分リロード」が確定済み。
  「自動時に自動ロード」もこの方針の流れで不要化された可能性が高い。
- 起点は「バックエンドの設定ダイアログ」を開く GUI コードと、
  そこで読み書きされる ConfigStore キーの探索。worker が grep で特定する。

## 想定される影響範囲

- GUI: バックエンド設定ダイアログを構成する Panel / Dialog / logic 関数
- 設定: `Settings` / `ConfigStore` の該当キー。既存ユーザ設定ファイルに
  キーが残っていても無視される動作になるか(後方互換シム不要方針)を要確認
- i18n: ja/en/zh/es 4 言語のラベル削除(AST 検査が走るのでカタログ整合を維持)
- テスト: ダイアログ smoke / 設定ラウンドトリップ / i18n 整合性 など
- ドキュメント: Class.md / Architecture.html の該当記述があれば現在形に整える
