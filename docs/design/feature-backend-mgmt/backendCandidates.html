backend 候補(レイヤ別ざっくり比較)

黄色背景 = 現 MVP。ローカル / クラウド。 「必要リソース」は概算(GB)で、推論時のピーク。クラウドはネット帯域とレイテンシが事実上の要件。 「認証」は事前準備の重さ — 不要 / API key / OAuth 等。 Phase F でどれを試すかの叩き台として使う。

1. Capture(音声取得)

名前形態認証必要リソース備考
soundcard (現 MVP) ローカル 不要 ~50MB Windows/Linux で loopback ◯。Mac は BlackHole 別途。pip 1 つで完結
sounddevice (PortAudio) ローカル 不要 ~50MB クロス OS。loopback は OS 設定依存。numpy 親和性が高い
PyAudio ローカル 不要 ~50MB PortAudio wrapper。メンテ落ち気味。新規採用は避ける
WASAPI Process Loopback ローカル 不要 ~50MB Windows 10+ の Per-app loopback。実装は pycaw/自前 ctypes
ScreenCaptureKit (macOS) ローカル 不要(権限ダイアログ) ~50MB macOS 12.3+ の per-app audio capture。Python ラッパは限定的
仮想デバイス (VB-CABLE/BlackHole) ローカル 不要 ~50MB OS にドライバを足してルーティングする workaround。Mac の loopback 代替

2. VAD(発話区切り検出)

名前形態認証必要リソース備考
Silero VAD (現 MVP) ローカル 不要 RAM ~0.3GB / モデル ~7MB ONNX。pip パッケージ同梱、DL 不要。CPU で十分速い。日本語/英語実績あり
WebRTC VAD ローカル 不要 RAM ~0.1GB C 実装。極軽量、ルールベース。長文発話の精度は Silero 劣後
pyannote.audio ローカル 不要(モデル DL) RAM ~2GB / VRAM ~1GB Transformer ベース。話者分離と一体運用可。重め
Picovoice Cobra ローカル アクセスキー RAM ~0.1GB 個人非商用は無料 tier。商用は要ライセンス。極軽量
energy-based 自作 ローカル 不要 ~0 RMS しきい値のみ。デバッグ/最終手段。精度低い

3. ASR(書き起こし)

名前形態認証必要リソース備考
faster-whisper (現 MVP) ローカル 不要 small: RAM 1.5GB / large: RAM 5GB / VRAM 1〜4GB CTranslate2 で int8 量子化。CPU でも実用。GPU で数倍速
openai-whisper (公式) ローカル 不要 RAM 2〜10GB / VRAM 2〜10GB PyTorch 直。faster-whisper より遅いが安定。研究/ベースライン向き
whisper.cpp ローカル 不要 RAM 0.5〜5GB C++ 実装。Python 経由は ctypes / バインディング。組み込み向き
vosk ローカル 不要 RAM 0.5〜2GB / モデル 50MB〜1GB Kaldi ベース。超軽量、低スペック PC で動く。精度は Whisper 劣後
OpenAI Whisper API クラウド API key ~50MB(HTTP client) $0.006/分。25MB/req 制限。レイテンシ実用域。large-v3 相当の精度
Deepgram (Nova-3) クラウド API key ~50MB $0.0058/分。WebSocket でリアルタイム。低レイテンシが売り
AssemblyAI クラウド API key ~50MB $0.37〜0.65/時間。話者分離・要約等の API も同居
Google Cloud STT クラウド サービスアカウント JSON ~100MB(SDK) $0.024/分(標準)。Chirp モデルでマルチ言語強い
Azure Speech クラウド サブスクリプションキー ~100MB(SDK) $1/時間相当。日本リージョン有り。継続発話で安価
AWS Transcribe クラウド IAM credentials ~100MB(boto3) $1.44/時間(標準)。IAM 設定が重め。ストリーミングあり

4. Translator(翻訳)

名前形態認証必要リソース備考
NLLB-200 distilled-600M (現 MVP) ローカル 不要 RAM 3GB / VRAM 2GB Meta、200 言語、MIT 互換。CPU 動作可、GPU で数倍速
NLLB-200 1.3B / 3.3B ローカル 不要 RAM 5〜13GB / VRAM 4〜10GB 同シリーズ大型版。精度↑、速度↓。GPU 推奨
OPUS-MT (Helsinki-NLP) ローカル 不要 RAM ~1GB(言語ペアごと) Marian ベース。言語ペアごとにモデル別。軽量だがペア数だけメモリ要る
M2M-100 (418M / 1.2B) ローカル 不要 RAM 2〜5GB / VRAM 1〜4GB Meta、100 言語。NLLB の前世代
DeepL API クラウド API key ~50MB 無料 500k 文字/月。Pro $5.49〜。日本語の品質はトップ評価
Google Cloud Translation クラウド サービスアカウント JSON ~100MB $20/百万文字。無料 500k 文字/月。安定の老舗
OpenAI GPT-4o-mini クラウド API key ~50MB $0.15/百万トーク入力。LLM 翻訳は文脈に強いが揺らぎ有り
Anthropic Claude (Haiku) クラウド API key ~50MB $0.8/百万トークン入力 (4.5 Haiku)。指示追従が安定
Azure Translator クラウド サブスクリプションキー ~100MB $10/百万文字。100 言語以上。F0 無料 tier 2M 文字/月

5. TTS(音声合成)

名前形態認証必要リソース備考
Windows SAPI / pyttsx3 (現 MVP) ローカル 不要 OS 内蔵 Windows 同梱。声質は最低限。Mac/Linux では NG。音節繰返しバグの暫定対処中
VOICEVOX ローカル 不要(別アプリ起動) RAM 1〜2GB 日本語特化、無料、商用利用可(キャラ別に規約)。HTTP API で叩く
Coqui TTS / XTTS-v2 ローカル 不要 RAM 3〜6GB / VRAM 4GB 多言語、声質クローニング可。MPL2.0。重め
Piper TTS ローカル 不要 RAM ~0.5GB ONNX、CPU 向け軽量。多言語、商用可。raspberrypi でも実用
edge-tts クラウド 不要(非公式) ~50MB Edge ブラウザ内蔵 API を叩く。無料だが MS 規約上グレー、本番運用不向き
ElevenLabs クラウド API key ~50MB $5/月〜。声質トップクラス、声クローニング可。月額制で従量も
OpenAI TTS クラウド API key ~50MB $15/百万文字。6 声、自然。日本語も実用
Google Cloud TTS クラウド サービスアカウント JSON ~100MB $16/百万文字 (Wavenet)。多声・SSML 対応。日本語声多数
Azure Speech (Neural Voice) クラウド サブスクリプションキー ~100MB $16/百万文字。Neural 声多数。F0 無料 tier あり
Amazon Polly クラウド IAM credentials ~100MB $4/百万文字 (Standard) / $16 (Neural)。多言語、SSML 対応

6. Output(音声出力)

名前形態認証必要リソース備考
soundcard (現 MVP) ローカル 不要 ~50MB Capture と同ライブラリ。複数サンプルレート切替容易
sounddevice ローカル 不要 ~50MB PortAudio、numpy 親和性◯。低レイテンシ調整に向く
PyAudio ローカル 不要 ~50MB PortAudio wrapper。メンテ落ち気味、新規採用非推奨
仮想オーディオデバイス出力 ローカル 不要 ~50MB VB-CABLE/VoiceMeeter/BlackHole の sink へ流して Zoom/Discord 等へ送る用途
ALSA / PulseAudio direct (Linux) ローカル 不要 ~10MB Linux 専用、低レイテンシ。Python bindings あり

Phase F 着手時の検討メモ

  • ASR で試すなら: OpenAI Whisper API(認証 1 つで動作確認しやすい、$ 安い)。Deepgram もリアルタイム性能の検証として面白い。
  • Translator で試すなら: DeepL API(日本語品質、無料 tier、認証 1 つ)。LLM 翻訳の比較対象として Claude Haiku もアリ。
  • TTS で試すなら: OpenAI TTS(認証 1 つ、自然、SAPI からの置換効果が明確)。VOICEVOX はローカル日本語の差し替えとして強い候補。
  • cloud backend は BackendRegistry.register(...., capabilities=BackendCapabilities(is_cloud=True, requires_credentials=True, service_name=..., terms_url=...)) で申告すれば、Phase D の同意ダイアログ + 認証フィールドが自動的に出る。
